For the following problem, generate black-box test cases using Decision Table method. Test the given program using these test cases.

The software enables a passenger to select a seat in an airplane. The plane has 25 rows (numbered 1 to 25). The rows 1 to 6 are reserved for Business Class passengers and has 4 seats in each row. Each seat has a seat type defined as follows:

W A A W

where 'W' stands for Window seat and 'A' stands for Aisle seat. The rest of the 19 rows are meant for Economy Class passengers and has 6 seats in each row. The seat arrangements in these rows are as follows:

W M A A M W

where, once again, 'W' stands for Window seat and 'A' stands for Aisle seat; 'M' stands for Middle seat.

 

For seat selection, the passenger must input three parameters:

  1. A ticket number (ranging from 1001 to 1138, both inclusive) -Ticket numbers 1001 to 1024 are reserved for Business Class passengers and the ticket numbers 1025 to 1138 are meant for Economy Class passengers.
  2. Row number (ranging from 1 to 25, both inclusive)
  3. Seat type (a character 'W', 'A' or 'M')

If the parameters are all valid and if the selected seat is not reserved before, the software assigns the seat for this ticket number. Some examples of invalid seat selections are as follows:

  • A Business Class passenger requests for a Middle seat.
  • Both Window seats in a particular row are already selected by other passengers and a passenger wants a Window seat on the same row.
  • An Economy Class passenger requests for a seat in one of the first 6 rows - remember that these rows are reserved for Business Class passengers.

The software displays the current seat assignments after all seat selection requests are done.
